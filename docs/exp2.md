### 2.2 将创建进程的过程修改为 fork/exec 的模式：

a. 实现两个 Xinu 系统调用 `int fork(void)` 和 `void exec(void*, pri16, char*, uint32, ...)`  
b. 当进程 A 在函数 foo()中调用 fork 后，创建出进程 B，在 fork 返回时，B 是就绪或正在运行状态（不是 create 创建的新进程那种挂起状态）  
c. fork 不接受参数，B 进程的栈大小可以设置默认值（如 64KB、8KB 等），也可以与父进程 A 保持一致  
d. fork 返回 0 时，表示在 B 进程的上下文中，此时如果执行 exec，则切换到指定的程序入口（如 sndA、sndB、delay_test 等）去执行；如果不执行 exec，则 B 进程直接沿着 foo() 函数中调用 fork 之后的代码继续执行（相当于复制了一个 A 进程）  
e. fork 返回正数（>0）时，表明在 A 进程的上下文中，返回值为 B 的进程 ID  
f. fork 返回负数时，表明创建子进程失败  
g. exec 函数接受至少 4 个参数：新进程入口函数地址（与 create 函数参数 funcaddr 一致）、新进程的优先级（create 函数的 priority 参数）、新进程的名称（create 函数的 name 参数）、传递给入口函数的参数的数目（create 函数的 nargs 参数），即：除了栈大小 ssize 参数外，create 函数的其他参数都出现在 exec 中  
h. exec 不会创建全新进程，而是在当前的 B 进程的上下文中设置栈并跳转到 funcaddr 执行；exec 正常执行的话不会返回到调用点  
i. fork 和 exec 的使用方式可参考课件第 4 章第 48 页，但父进程无需使用 waitpid 之类的方式等待子进程结束  
j. 在 `xsh_lab2.c` 中实现个测试函数  
k. 测试函数 1 的架构如下（只使用 fork，不使用 exec），父子进程从 fork 返回后都输出当前正在执行的进程的 PID（返回值和 currpid 相对对照）和进程名称。（自行调整代码，以确保结果正确并排除创建子进程失败的影响）

```c
int pid = fork();
printf("curr proc: %d(%d), %s\n", pid, currpid, 进程名称);
```

l. 测试函数 2 的结构如下（使用 fork 和 exec），其中，exec 的目标程序中至少应输出当前进程的 pid 和进程名，最后一个输出 currpid 的行用于检查子进程是否从 exec 返回了（正常来说，子进程是不会返回然后执行该 printf 的）。

```c
int pid = fork();
if (子进程) {
    exec(目标程序, 其他参数) // 目标程序需自行提供
} else if (父进程) {
    printf("parent proc: %d(%d), %s\n", pid, currpid, 进程名称);
}
printf("currpid = %d\n", currpid);
```
